# Sprint 1: Core UI Foundations

## Priority

`P0` (MVP-critical)

## Objective

Implement the calm, tokenized visual baseline and core UI shells so all
later features plug into a stable design system.

## In Scope

- Theme tokenization and Tailwind/shadcn semantic class alignment
- Auth layout and main app shell
- Dashboard and deck management base components

## Out of Scope

- SRS scheduling persistence
- Anki import processing
- Deployment setup

## Dependencies

- None (starting sprint)

## Exit Criteria

- Design tokens mapped and used by default component wrappers
- `/login`, `/dashboard`, and deck views share consistent
	shell/layout primitives
- Dashboard can render deck cards and create-deck affordance with the
	defined style language
